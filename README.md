# RecoilLab

Веб-приложение на Django для расчёта откатной системы с поддержкой нескольких магнитных тормозов, сохранением результатов, интерактивной визуализацией через Plotly и сравнением выполненных расчётов.  
Текущая версия интерфейса включает страницу создания нового расчёта с несколькими карточками тормозов, страницу результатов с интерактивными графиками по общему процессу, фазе отката и фазе наката, а также страницу сравнения расчётов. :contentReference[oaicite:0]{index=0}

## Назначение

Проект предназначен для инженерного расчёта откатной системы с учётом:

- временного поведения перемещения, скорости и ускорения;
- разделения процесса на фазу отката и фазу наката;
- работы одного или нескольких магнитных тормозов;
- анализа силовых составляющих в течение расчёта;
- сохранения и повторного использования параметров ранее выполненных расчётов.

Приложение ориентировано на последовательную работу инженера: ввести параметры, выполнить расчёт, изучить графики, сохранить отчёт, затем при необходимости подставить параметры предыдущего расчёта в новый или сравнить два запуска между собой.

## Основные возможности

### 1. Создание нового расчёта

На главной странице доступны:

- ввод общих параметров расчёта;
- загрузка входного файла;
- добавление произвольного количества карточек магнитных тормозов;
- копирование существующей карточки тормоза;
- удаление карточки тормоза;
- автоматическая блокировка интерфейса во время выполнения расчёта с отображением состояния «Идёт расчёт». :contentReference[oaicite:4]{index=4}

### 2. Работа с несколькими магнитными тормозами

Каждый тормоз задаётся отдельным набором параметров.  
Карточки тормозов можно:

- добавлять;
- копировать;
- удалять;
- автоматически перенумеровывать после изменений. :contentReference[oaicite:5]{index=5}

### 3. Страница результата расчёта

Страница результата содержит:

- итоговые численные показатели;
- статус завершения расчёта;
- предупреждения;
- интерактивные графики;
- исходные параметры расчёта;
- параметры всех магнитных тормозов;
- возможность подставить параметры расчёта в новую форму;
- возможность удалить расчёт вместе с его файлами. :contentReference[oaicite:6]{index=6}

### 4. Интерактивные графики

Графики строятся в Plotly и отображаются прямо на странице результата.  
В текущей версии формируются:

#### Общие графики

- перемещение от времени;
- скорость и ускорение от времени;
- скорость от перемещения;
- распределение сил от времени;
- магнитные силы от скорости.

#### Графики фазы отката

- перемещение;
- скорость и ускорение;
- движущая и общая сила;
- распределение сил.

#### Графики фазы наката

- перемещение;
- скорость и ускорение;
- распределение сил.  
На графике распределения сил для наката также отображается суммарная сила.

### 5. Сравнение расчётов

Система поддерживает режим сравнения двух ранее выполненных расчётов:

- сравнение итоговых таблиц результатов;
- параллельный просмотр однотипных графиков рядом друг с другом.  
Эта возможность реализуется на отдельной странице сравнения.

### 6. Экспорт в Excel

Для каждого расчёта формируется Excel-отчёт с данными и графиками.  
Ссылка на скачивание отчёта доступна на странице результата. :contentReference[oaicite:11]{index=11}

## Архитектура решения

Проект построен по классической схеме Django-приложения:

- **формы** — ввод и валидация параметров расчёта;
- **представления (`views`)** — приём данных, запуск расчёта, сохранение результатов, подготовка страниц;
- **модели** — хранение параметров расчётов, конфигураций магнитных тормозов и ссылок на артефакты;
- **сервисы** — отдельные модули для расчёта динамики, обработки магнитных тормозов, построения графиков и формирования Excel-отчёта;
- **шаблоны** — HTML-интерфейс страниц ввода, результатов и сравнения;
- **файловое хранилище** — отдельная директория для артефактов каждого расчёта.

Такой подход разделяет пользовательский интерфейс, вычислительную логику и файловую инфраструктуру, что упрощает сопровождение проекта.

## Установка и запуск

Клонирование проекта

```text
git clone <URL_РЕПОЗИТОРИЯ>
cd <ПАПКА_ПРОЕКТА>
```

Создание виртуального окружения

Windows

```text
python -m venv .venv
.venv\Scripts\activate
```

Linux / macOS

```text
python3 -m venv .venv
source .venv/bin/activate
```

1. Установка зависимостей

```text
pip install -r requirements.txt
```

1. Применение миграций

```text
python manage.py makemigrations
python manage.py migrate
```

1. Создание администратора

```text
python manage.py createsuperuser
```

1. Запуск сервера разработки

```text
python manage.py runserver
```

После запуска приложение обычно доступно по адресу:

```text
http://127.0.0.1:8000/
```

## Описание интерфейса

### Главная страница

Главная страница содержит:

- блок общих параметров;
- блок магнитных тормозов;
- список последних расчётов.

Во время отправки формы появляется overlay с сообщением «Идёт расчёт», а повторный запуск блокируется.

### Страница результата

На странице результата доступны разделы:

- итоговые результаты;
- статус расчёта;
- общие графики;
- графики отката;
- графики наката;
- исходные данные расчёта.

Большие разделы можно сворачивать и разворачивать.

### Страница сравнения

Страница сравнения показывает два расчёта параллельно:

- таблицы;
- графики одинакового типа.

## Графики

### Общие графики результатов

В текущей версии доступны:

- x(t) — перемещение от времени;
- v(t) + a(t) — скорость и ускорение от времени;
- v(x) — скорость от перемещения;
- распределение сил от времени;
- магнитные силы от скорости.

На графике магнитных сил от скорости отображаются:

- сила каждого тормоза;
- суммарная магнитная сила.

### Графики результата фазы отката

В фазе отката отображаются:

- перемещение;
- скорость и ускорение;
- движущая и суммарная силы;
- распределение сил.
- Графики фазы наката

В фазе наката отображаются:

- перемещение;
- скорость и ускорение;
- распределение сил.

На графике распределения сил для наката дополнительно отображается суммарная сила.

### Особенность графика v(t) + a(t)

График скорости и ускорения строится с двумя осями, при этом диапазоны подбираются так, чтобы нулевые линии обеих величин находились на одной горизонтали. Это улучшает визуальную интерпретацию переходов через ноль.

## Экспорт результатов

Для каждого расчёта формируется Excel-отчёт.
Отчёт включает:

- summary-лист с итоговыми показателями;
- основной лист с табличными данными;
- отдельные листы с графиками.

Excel-файл можно скачать со страницы результата.

### Хранение расчётов и файлов

Каждый расчёт сохраняется в базе данных и сопровождается файлами в

```text
media/reports/<имя_расчёта>_<id>/.
```

Обычно в этой директории хранятся:

- HTML-фрагменты графиков;
- Excel-отчёт.

Такой подход позволяет:

- удобно удалять расчёт вместе с файлами;
- не смешивать артефакты разных запусков;
- повторно открывать сохранённые графики в интерфейсе.

## Администрирование

Через Django admin доступны:

- просмотры запусков расчётов;
- параметры магнитных тормозов;
- ссылки на графики и отчёты;
- итоговые параметры расчётов.

Админка полезна для:

- проверки корректности сохранения данных;
- ручного анализа записей;
- диагностики проблем.

## Ограничения текущей версии

Текущая версия имеет ряд практических ограничений:

- интерфейс ориентирован на локальное или стендовое использование;
- входной файл должен соответствовать ожидаемому формату расчётных характеристик;
- система сравнения зависит от сохранённых расчётов, а не от временных несохранённых сценариев;
- часть инженерной терминологии и структуры отчёта может уточняться по мере развития предметной области.
